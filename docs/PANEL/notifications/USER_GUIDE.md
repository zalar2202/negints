# Notification System – User Guide

## Getting started

1. Log in to the panel.
2. When prompted, click **Allow** for notifications.
3. No further setup; you’ll receive in-app and push notifications.

---

## Viewing notifications

**Dropdown (quick view):**

- Click the bell icon (top-right). Red badge = unread count.
- See the latest 5; click one to open its action or mark it read.
- **View all notifications** → full page at **/panel/notifications**.

**Full page:**

- Sidebar → **Notifications**, or from the dropdown → **View all notifications**.
- Tabs: All, Unread, Read. Filter by type (Success, Info, Warning, etc.).

---

## Managing notifications

| Action | How |
|--------|-----|
| Mark one read | Click the notification (dropdown or list) |
| Mark all read | Notifications page → **Mark all read** |
| Delete one | Notifications page → **Delete** next to the item → confirm |
| Delete all read | Notifications page → **Delete all read** → confirm |

---

## Types and content

- **Success** – e.g. account activated  
- **Info** – general updates  
- **Warning** – important notices  
- **Error** – issues that need attention  
- **System** – system messages  
- **Admin** – messages from admins  

Each notification has title, message, time, and optionally a link/action.

---

## When the panel is open vs closed

- **Panel open:** Toast (top-right), badge updates; list updates on next poll (~30s) or when you open the dropdown.
- **Panel closed / background:** OS push notification; click to open the panel (and optional action URL).
